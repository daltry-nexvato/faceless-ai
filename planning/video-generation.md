# Video Generation Module — Finalized Spec (Audited)

## What This Module Does
Generates short AI video clips (3-10 seconds each) that can be used as visual slots within scenes. These are NOT the final rendered video — that's the Editing module's job. This module produces individual motion clips that sit alongside still images as visual options.

---

## When Video Clips Are Used
- **Optional, not default.** Most faceless videos use still images with Ken Burns (pan/zoom) effects. AI video clips are a premium upgrade.
- **Typical use:** 3-5 video clips per 10-minute video, mixed with 20+ still images
- **High-credit cost:** See credit scaling below (intentionally expensive — these are the costliest AI operations)
- **Tier gated:** Starter tier does NOT have access. Creator+ only.

---

## Video Generation via fal.ai

All video generation goes through fal.ai's unified API, same as images.

### Model Selection Logic
- **Draft quality:** Cheapest model (Wan 2.1 or Luma) — for preview/iteration
- **Standard quality:** MiniMax or Kling — balanced quality/cost
- **Premium quality:** Kling 1.6 or Runway Gen-3 — best results
- Admin configurable: can swap default models as new ones launch on fal.ai

## Two Ways to Generate Video Clips

### 1. Text-to-Video (prompt only)
- User provides a text description, AI generates the clip from scratch
- Prompt built from: scene script + Style visual direction + user customization
- Example: "Aerial drone shot of a dense tropical rainforest at sunrise, misty, cinematic, slow camera movement"

### 2. Image-to-Video (still image + motion) — RECOMMENDED DEFAULT
- Takes an existing still image (AI-generated, stock, or uploaded) and animates it
- Much more predictable results — you know what the starting frame looks like
- Adds motion: camera pan, zoom, character movement, environmental effects
- Example: Take a generated image of a historical scene → animate with subtle movement
- **Motion prompt construction:** System builds the motion description from: the scene script (what's happening in the scene), the camera movement setting (user's choice), the motion amount setting (low/medium/high), and the Style visual direction. Example: scene says "explorer walks through ancient ruins" + user picks "Pan Right" + Medium motion → motion prompt becomes "Slow pan right revealing ancient stone ruins, a figure walking through, medium movement, cinematic lighting."
- **Aspect ratio handling:** If the source image doesn't match the project aspect ratio, system auto-crops or outpaints the image to match BEFORE sending to the video model. Uses the same tools from the Images module (Sharp crop, fal.ai outpainting). User gets a warning: "This image will be cropped to fit 16:9 before animation."

## How It Fits in the Pipeline

### During Visuals Step (after voiceover is approved):
1. Each scene has visual slots to fill
2. User picks source per slot: AI Image, Stock Image, Stock Video, Upload, **or AI Video Clip**
3. If AI Video Clip selected → user chooses text-to-video or image-to-video
4. For image-to-video: user first selects/generates the base image, then animates it

### Stock Video Clips vs AI Video Clips:
- **Stock video clips** (from Pexels/Pixabay) are handled by the **Images module** — they come from the same stock search pipeline, cost 0 credits
- **AI-generated video clips** are handled by **this module** — generated by fal.ai video models, cost credits
- The visual slot picker makes this clear: "Stock Video" appears alongside "Stock Image" (both from Images module), "AI Video Clip" is a separate option (from this module)

### Timing Connection to Voiceover:
- Voiceover scene timing tells us how long each scene lasts
- A 5-second video clip in a 45-second scene means it fills part of the scene
- The Editing module handles stitching clips + images together to fill the full scene duration

### Visual Anchor Support:
- For image-to-video, the base image can be a Visual Anchor image (consistent character/object)
- The video model then animates that consistent character — maintaining visual continuity

---

## Preview & Approval Flow

1. User requests a video clip → job created (IN_PROGRESS, shows in job queue)
2. Generation takes 1-5 minutes depending on model
3. Preview plays inline in the visual slot (no watermark on previews — only the final rendered video gets watermarked for trial users)
4. User can: **Approve**, **Regenerate** (new attempt, costs credits again), or **Replace** (switch to a different visual source)
5. Approved clips are stored in S3 and locked to that visual slot

---

## Audio Handling
- **All audio is stripped from generated video clips by default.** Voiceover is the primary audio track; we don't want competing audio from AI-generated clips.
- If a user wants ambient sound for a scene (e.g., rain, crowd noise), they generate it separately through the ElevenLabs Sound Effects API (voiceover module) and layer it in the Editing module.
- This keeps the audio pipeline clean and predictable.

---

## NSFW & Content Moderation
- Same pipeline as images: every generated video clip runs through fal.ai NSFW detection
- Blocked clips show a friendly message + option to adjust prompt and retry
- Additional check: video clips are sampled at 3 frames (start, middle, end) for moderation

---

## Cost Controls
- **Credit costs:** See credit scaling table above (linear: 15 credits/sec Draft, 30 credits/sec Standard/Premium)
- Users see estimated credit cost before generating
- Draft quality available for cheaper iteration
- Max concurrent video generation jobs per tier (Starter: not available, Creator: 2, Pro: 5, Agency: 10)

---

## Storage & Media Library

### Storage:
- Generated clips stored in S3 as MP4 (H.264)
- Intermediate clips deleted after 30 days if project is archived
- Final clips preserved as long as the project exists

### Media Library Metadata:
- Clips available in Media Library for reuse across projects
- Metadata per clip: project, channel, scene, duration, source type (text-to-video or image-to-video), source image ID (if image-to-video), model used, prompt used, motion settings, camera movement setting, quality tier, generation timestamp

---

## Cascade Invalidation Warning

Video clips are the most expensive visual asset. When cascade invalidation (from platform-systems.md) would affect scenes that contain AI video clips, the system shows a special cost warning:

- "This change will invalidate 3 AI video clips. Regenerating them will cost ~450 credits. Still images in affected scenes cost ~45 credits to regenerate."
- Gives the user a chance to reconsider before burning expensive credits
- User can choose: regenerate all, regenerate selectively, or swap video clips for cheaper still images

---

## Error Handling
- If generation fails: auto-retry once with same parameters
- If retry fails: notify user with "Generation failed — try a different prompt or switch to a still image"
- If fal.ai video service is down: queue the request and process when service recovers (video clips are never urgent — they're a premium extra)
- Timeout: if generation exceeds 10 minutes, mark as failed and offer retry

## Dependencies
- **fal.ai** — all video model inference
- **Images module** — image-to-video uses images as input; stock video clips live in Images module
- **Voiceover module** — scene timing data determines how clips fit into scenes
- **Editing module** — stitches video clips + still images into the final timeline; receives clips with audio stripped
- **Media Library** — stores and indexes approved video clips with full metadata for reuse
- **Billing/Credits** — linear credit scaling (15-30 credits/sec by quality), tier-gated (Creator+)
- **Platform Systems** — cascade invalidation shows credit cost warning for affected video clips
